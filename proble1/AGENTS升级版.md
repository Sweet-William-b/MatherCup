- **问题一第二子问题对应升级版模型B**：只允许使用一种车型，将全部货物运输完成且所用车辆数最少。

- 比单纯分层法更充分地利用不规则空间
- 能输出具体坐标与支撑关系
- 在保留块化降维优势的同时提升单车装载质量

当前推荐方法仍然**不追求严格全局最优**。  
它属于更强的工程化近似模型，重点是：

- 模型表达比基线更真实
- 装载方案比基线更灵活
- 代码实现仍有较强可落地性

你的输出必须覆盖以下内容：

你的任务重点不是只给抽象思路，而是把方案写到“**别人可以据此写论文并继续实现升级版代码**”的程度。

## 一、问题理解

问题一可拆解为两个相互关联但目标不同的子问题：

1. **问题一第一子问题 / 升级版模型A**：固定单一车型，构造三维装载策略，使单车装载效果最佳。
2. **问题一第二子问题 / 升级版模型B**：若只允许使用一种车型，将附件1中的全部货物装完，并使车辆数最少。

其中，升级版模型A的任务是：

> 在给定车型的前提下，考虑尺寸、重量、堆叠、方向、稳定性和安全间隙等约束，选择并摆放货物，使单车的“满容”和“满载”尽量同时达到最大。

升级版模型B的任务是：

> 在只允许使用一种车型的前提下，将全部货物划分到若干辆同型车辆中，并在满足三维装箱约束的前提下，使所需车辆总数最少。

本 Agent 将升级版模型A与升级版模型B分开表述：

- 升级版模型A重点解决“单车怎么装得更好”
- 升级版模型B重点解决“全局怎么分配更省车”

## 二、为什么选这条升级主线

这道题不适合直接使用完整的全三维大规模整数规划，也不建议继续停留在纯分层构造。当前推荐升级路线的原因如下：

### 1. 货物种类少、同类数量多，仍然适合块化

附件1中的货物类型有限，但同类件数较多。  
若完全按逐件三维搜索处理，状态空间会膨胀得很快。  
因此仍需要保留“块化”这一步，用规则块或超物品先压缩规模。

### 2. 单纯分层会限制三维空间利用率

分层法便于实现，但会天然偏向等高铺排。  
对于边角空间、错缝空间、局部凹槽和高低错落区域，分层法往往难以充分利用。  
极点法和最大空箱法能更自然地描述真实三维可用空间，因此更适合作为升级版模型A的核心放置机制。

### 3. 题目有稳定性要求，必须显式刻画支撑关系

本题不是单纯的体积装箱，还涉及：

- 易碎件不能承压
- 定向件不能任意翻转
- 上下层需要稳定支撑
- 顶部要预留安全间隙

若只依靠“整层支撑整层”的规则，会损失大量可行装法。  
因此升级版模型A引入**支撑图约束**，将支撑关系细化到块与块之间。

### 4. 需要输出具体坐标，所以空间状态必须可回填

题目最终要求输出每件货物的摆放坐标和姿态方向。  
极点法和最大空箱法都是围绕具体坐标进行空间更新，因此更适合生成逐件可执行方案。

### 5. 多车问题仍然最适合模式法处理

即便单车模型升级，多车问题的核心仍然不是“每辆车各自装得多满”，而是“多辆车如何协同装完全部货物”。  
因此升级版模型B仍然采用模式法：

- 升级版模型A负责造模式
- 升级版模型B负责选模式

## 三、核心建模假设

为保证建模与实现一致，默认采用以下假设：

- 尺寸单位统一为厘米，重量单位统一为千克。
- 车厢有效高度统一按 `H_eff = H - 3` 处理，用于满足顶部 `3cm` 安全间隙要求。
- 所有货物均按正交方式摆放，货物边与车厢边平行。
- 标准件允许所有合法正交姿态。
- 定向件只允许题目给定姿态。
- 易碎件不承担上层载荷，不允许成为支撑图中的承压父节点。
- 允许采用规则块或超物品作为放置单元，但最终必须回填到逐件坐标。
- 非底层块必须满足最小支撑覆盖比例约束，必要时可加入重心投影约束。
- 当前方法追求高质量可行解，不宣称全局最优解。

其中支撑覆盖比例可记为 `rho_min`，通常取 `0.8` 到 `1.0` 的区间，由实现阶段调参。

## 四、升级版模型A：问题一第一子问题

### 4.1 要解决什么

升级版模型A对应问题一第一子问题，核心任务不是“把全部货都运完”，而是：

> 在固定一种车型的前提下，从全部待运货物中选出适合的一部分，并给出它们在单辆车内的三维摆放方式，使单车的空间利用率和载重利用率尽量同时高。

换句话说，升级版模型A研究的是“**单车怎么装得更强、更灵活、更接近真实三维装载**”。

### 4.2 建模思路

升级版模型A采用一种**块化 + 三维空间状态更新 + 支撑图约束**的结构化近似模型，其核心流程为：

1. 将单件货物按合法姿态组合成候选块。
2. 维护当前车厢中的极点集合和最大空箱集合。
3. 在候选可放位置上选择最优块进行放置。
4. 每放置一个块，就更新空间状态与支撑图。
5. 当初始装载完成后，再对低效率区域执行局部重构修补。

因此，升级版模型A的核心逻辑是：

> 单件先块化，块在三维空间中按极点或空箱动态放置，并用支撑图约束保证稳定性，最后通过局部重构进一步提升利用率。

### 4.3 为什么这样建模

升级版模型A这样建模有四个直接原因：

1. 货物种类少但数量多，适合用规则块压缩规模。
2. 题目最终要求输出具体坐标，极点法和最大空箱法天生适合坐标回填。
3. 题目存在稳定性要求，支撑图比单纯层间支撑更细。
4. 相比纯分层法，该模型更能利用边角和不规则空位。

因此，升级版模型A应明确写成：

> 一个基于块化、极点/最大空箱空间更新和支撑图约束的单车三维装箱近似模型。

这里需要明确承认：

- 该模型是高质量近似建模
- 不是全三维精确整数规划
- 重点是得到更强的单车可行解和更完整的坐标方案

### 4.4 数学模型

#### 1. 基本符号

设：

- 车型内尺寸为 `L, W, H`
- 有效高度为 `H_eff = H - 3`
- 车辆额定载重为 `Q`
- 货物类型集合为 `I`
- 候选块集合为 `B`
- 已放置块实例集合为 `K`
- 极点集合为 `E`
- 最大空箱集合为 `M`

对候选块 `b in B`，定义：

- `l_b, w_b, h_b`：块的长宽高
- `q_b`：块重量
- `v_b`：块体积
- `a_ib`：块 `b` 中包含类型 `i` 的货物件数
- `support_b`：块 `b` 是否允许承压

对已放置块实例 `k in K`，定义：

- `(x_k, y_k, z_k)`：块左下后角坐标
- `(dx_k, dy_k, dz_k)`：块放置后的外接尺寸
- `o_k`：块姿态编码
- `u_k in {0,1}`：块实例是否被放入车辆

对支撑关系，定义：

- `s_jk in {0,1}`：若块 `j` 为块 `k` 的支撑者，则 `s_jk = 1`
- `A_jk`：块 `j` 顶面与块 `k` 底面的投影重叠面积
- `S_k`：块 `k` 的底面积

决策层面可将变量理解为：

- 上层决定选择哪些块进入装载组合
- 下层决定这些块放在什么位置、以什么姿态摆放
- 支撑图刻画块与块之间的承压关系

#### 2. 目标函数

升级版模型A仍以综合满载率最大化为目标：

```text
max lambda
```

并约束：

```text
lambda <= (sum(v_b * u_b)) / (L * W * H_eff)
lambda <= (sum(q_b * u_b)) / Q
```

这样定义的原因仍然是：

- 只看体积，会偏向泡货
- 只看重量，会偏向重货
- 题目要求“满容”和“满载”尽量同时达到，因此要用二者较小值衡量综合效果

因此，`lambda` 可以理解为：

> 体积利用率与载重利用率中的较小者。

#### 3. 主约束

升级版模型A的主约束包括：

```text
sum(a_ib * u_b) <= n_i,   for all i in I
sum(q_b * u_b) <= Q
x_k + dx_k <= L
y_k + dy_k <= W
z_k + dz_k <= H_eff
```

除此之外，还包括以下三类关键约束：

**（1）不重叠约束**

任意两个已放置块 `j, k` 满足三维上至少在一个方向分离。  
实现时通常不直接写完整大规模整数约束，而是由极点法或最大空箱法的可行性检查来保证。

**（2）姿态合法约束**

- 标准件允许所有合法正交姿态
- 定向件只允许规定姿态
- 姿态一旦选定，则相应 `dx_k, dy_k, dz_k` 被确定

**（3）支撑图约束**

对任一非底层块 `k`，要求：

```text
sum(A_jk * s_jk) >= rho_min * S_k
```

并满足：

```text
s_jk = 0,   if block j is fragile or not supportable
```

必要时还可补充：

- 底面重心投影落在支撑区域内
- 顶层易碎件只能作为终端节点，不能继续向上承压

#### 4. 约束的现实含义

升级版模型A中的每类约束都对应真实装载要求：

- 边界约束：货物不能超出车厢
- 数量约束：不能虚构货物
- 载重约束：不能超过额定载重
- 不重叠约束：保证物理上摆得进去
- 姿态约束：保证满足题目给定方向条件
- 支撑图约束：保证装载过程稳定可执行
- 安全间隙约束：满足顶部预留空间要求

所以升级版模型A不是抽象的“数学拼箱”，而是更接近实际仓配作业的三维装载模型。

### 4.5 求解方法

升级版模型A建议固定按下面七步描述：

1. 货物展开与合法姿态生成
2. 候选块生成
3. 初始极点与最大空箱初始化
4. 三维放置搜索与空间状态更新
5. 支撑图判定与稳定性校验
6. 局部重构修补
7. 结果输出与校验

更具体地说：

- `块` 用来压缩规模
- `极点` 用来生成有限可放位置
- `最大空箱` 用来筛选当前可利用三维空间
- `支撑图` 用来保证稳定性
- `局部重构修补` 用来提升后期残余空间利用率

#### 1. 极点法与最大空箱法的混合使用

推荐采用混合式空间状态管理：

- 极点法负责生成候选放置坐标
- 最大空箱法负责判断某个块是否能嵌入当前剩余空间

这样做的优点是：

- 纯极点法候选点多，但筛选空间略粗
- 纯最大空箱法空间表达准，但候选动作不够紧凑
- 混合使用能兼顾效率与质量

#### 2. 放置评分函数

对每个可行动作 `(块, 姿态, 极点/空箱)`，建议构造综合评分：

```text
Score = alpha1 * Δlambda
      + alpha2 * 空间贴合度
      + alpha3 * 支撑质量
      + alpha4 * 重量利用提升
      - alpha5 * 碎片化惩罚
      - alpha6 * 不稳定风险
```

这一步是升级版模型A的核心启发式。

#### 3. 局部重构修补

升级版模型A不再只采用“顶层追加修补”，而建议采用：

- 删除利用率低的若干块
- 释放局部空域
- 对释放区域重新执行三维放置

可将其写为：

> 对初始解中的低效率区域执行小规模破坏-重构，通过重新生成局部极点与空箱状态，提升中后期残余空间利用率。

`LNS/ALNS` 在该模型中仍可作为可选改进模块，但不应替代模型主线本身。

### 4.6 输出格式

升级版模型A最终建议输出三类结果：

## 五、升级版模型B：问题一第二子问题

### 5.1 要解决什么

升级版模型B对应问题一第二子问题，核心任务不是“某一辆车怎么装最好”，而是：

> 在只允许使用一种车型的前提下，把全部货物运输完成，并使总车辆数最少。

换句话说，升级版模型B研究的是“**全局怎么分配更省车**”。

### 5.2 为什么不能直接重复升级版模型A

升级版模型B不能简单理解为“把升级版模型A跑很多遍”，原因是：

1. 升级版模型A是单车局部优化，升级版模型B是多车全局优化。
2. 单车装得好的方案重复使用后，可能会耗尽某些关键货类，导致后续车辆更难组合。
3. 多车问题关键在于不同车辆之间的互补，而不是每辆车都各自最满。

因此，升级版模型B仍需独立建模。

### 5.3 建模思路

升级版模型B采用一种与升级版模型A衔接的模式法思路：

1. 调用升级版模型A生成多种单车可行装载模式。
2. 将“一辆车的一种完整装法”抽象为一个装载模式。
3. 在模式库中选择尽量少的模式，使全部货物被刚好覆盖。
4. 对主问题求解后暴露出的残量需求，再定向生成残量模式修补。

因此，升级版模型B采用：

> **装载模式集合划分模型 + 自适应模式库生成 + 整数主问题搜索 + 残量模式修补**

其中，自适应的含义是：

- 模式库不是一次性固定死的
- 主问题当前的缺口会反过来指导新模式生成

### 5.4 主问题

设：

- `I`：货物类型集合，`i in I`
- `P`：可行装载模式集合，`p in P`
- `n_i`：第 `i` 类货物总需求件数
- `a_ip`：模式 `p` 中装入的第 `i` 类货物件数
- `y_p`：模式 `p` 被使用的次数

其中每个模式 `p` 表示：

- 一辆同型号车辆的一种完整三维装载方案
- 该方案已经满足尺寸、重量、姿态、支撑与稳定性等所有约束

主问题目标函数：

```text
min sum(y_p),  p in P
```

约束：

```text
sum(a_ip * y_p) = n_i,   for all i in I
y_p in Z_>=0,            for all p in P
```

这三个式子的含义是：

- 每选一次模式，就相当于用一辆车
- 所有模式中的各类货物数量加总后，必须刚好等于总需求
- 模式使用次数必须是整数，因为不存在半辆车

### 5.5 模式生成

升级版模型B中的模式全部由升级版模型A生成。  
模式来源建议包括：

- 综合满载率较高的平衡模式
- 重货优先模式
- 泡货优先模式
- 易碎件后置模式
- 支撑稳定优先模式
- 残量定向修补模式

每个模式至少记录：

- 各类货物件数
- 单车逐件坐标方案
- 支撑图摘要
- 体积利用率
- 载重利用率
- 综合满载率

### 5.6 主问题求解策略

升级版模型B的求解过程建议按下面顺序写：

1. 固定一种车型。
