## 一、问题理解

问题一可拆解为两个相互关联但目标不同的子问题：

1. **问题一第一子问题 / 模型A**：固定单一车型，构造三维装箱策略，使单车装载效果最佳。
2. **问题一第二子问题 / 模型B**：若只允许使用一种车型，将附件1中的全部货物装完，并使车辆数最少。

其中，模型A的任务是：

> 在给定车型的前提下，考虑尺寸、重量、堆叠、方向、稳定性和安全间隙等约束，选择并摆放货物，使单车的“满容”和“满载”尽量同时达到最大。

模型B的任务是：

> 在只允许使用一种车型的前提下，将全部货物划分到若干辆同型车辆中，并在满足三维装箱约束的前提下，使所需车辆总数最少。

本 Agent 将模型A与模型B分开表述：

- 模型A重点解决“单车怎么装最好”
- 模型B重点解决“全局怎么分配最省车”

---

## 二、为什么选这条方法主线

这道题不适合直接使用纯整数规划，也不建议只用简单贪心。当前推荐方法的原因如下：

### 1. 货物种类少、同类数量多，适合块化

附件1中的货物类型有限，但同类件数较多。  
若直接逐件搜索三维坐标，变量数量和不重叠关系会迅速膨胀。  
将若干同类、同姿态货物先组合成规则块，可以显著压缩规模。

### 2. 支撑、堆叠和稳定性要求强，适合分层

题目不是普通体积装箱，还要求：

- 易碎件不能被压
- 定向件不能任意翻转
- 上下层需要稳定支撑
- 顶部要预留安全间隙

直接在完整三维空间中同时处理这些约束会非常复杂。  
将车厢沿高度方向拆成若干层，再按层叠加，更容易兼顾几何可行性与稳定性。

### 3. 需要输出坐标与可执行方案，结构化近似更容易落地

题目最终要求输出每件货物的摆放坐标和姿态方向。  
因此方法不能只停留在总体积或总重量层面，必须能够回填到具体装载位置。

“块化 + 分层”的结构化方法既便于生成具体坐标，也便于和后续程序实现对接。

### 4. 分层模型本身会遗漏零散空位，所以必须补局部插入

分层法擅长铺出规则大结构，但容易留下顶面和边角零碎空间。  
因此模型A不能只做“选层结束”，还必须增加“局部插入修补”步骤，用来吸收剩余少量小件。

### 5. 多车问题更适合模式法，而不是多车三维坐标一体化

模型B的关键不是“每辆车单独装得多满”，而是“多辆车如何协同装完全部货物”。  
因此更自然的做法是：

- 先由模型A生成单车可行装载模式
- 再由模型B在模式层面做全局选择

这比直接建立“大一统多车三维坐标模型”更稳，也更适合当前题目规模。

## 三、核心建模假设

为保证建模与实现一致，默认采用以下假设：

- 尺寸单位统一为厘米，重量单位统一为千克。
- 车厢有效高度统一按 `H_eff = H - 3` 处理，用于满足顶部 `3cm` 安全间隙要求。
- 所有货物均按正交方式摆放，货物边与车厢边平行。
- 标准件允许所有合法正交姿态。
- 定向件只允许题目给定姿态。
- 易碎件不承担上层载荷，默认只能位于最上方或作为不承压层处理。
- 分层主模型中，每层优先由等高块组成，以增强稳定性和简化层间组合。
- 当前方法追求高质量可行解，不宣称全局最优解。

## 四、模型A：问题一第一子问题

### 4.1 要解决什么

模型A对应问题一第一子问题，核心任务不是“把全部货都运完”，而是：

> 在固定一种车型的前提下，从全部待运货物中选出适合的一部分，并给出它们在单辆车内的摆放方式，使单车的空间利用率和载重利用率尽量同时高。

换句话说，模型A研究的是“**单车怎么装最好**”。

### 4.2 建模思路

模型A采用一种**结构化近似装箱模型**，其思路不是直接对每件货物做完整三维坐标优化，而是分三层处理：

1. 先把单件货物按合法姿态组合成候选块。
2. 再把候选块排布为候选层。
3. 然后在层之间做组合选择，形成单车初始装载方案。
4. 最后对剩余未装入的小件货物做局部插入修补。

因此，模型A的核心逻辑是：

> 单件先块化，块再成层，层间组合成单车方案，剩余小件通过局部插入吸收。

这样做的目的不是追求理论上最精确的三维整数规划，而是在题目规模和实现成本之间取得平衡。

### 4.3 为什么这样建模

模型A这样建模有三个直接原因：

1. 货物类型少但数量多，适合用规则块压缩规模。
2. 题目支撑、堆叠和方向约束强，适合按层组织装载。
3. 题目最终要求输出具体坐标，结构化建模更容易落地到程序。

因此，模型A应明确写成：

> 一个基于块化、分层和局部修补的单车三维装箱近似模型。

这里需要明确承认：

- 该模型是高质量近似建模
- 不是全三维精确整数规划
- 重点是得到稳定、可执行、可输出坐标的装载方案

### 4.4 数学模型

#### 1. 基本符号

设：

- 车型内尺寸为 `L, W, H`
- 有效高度为 `H_eff = H - 3`
- 车辆额定载重为 `Q`
- 货物类型集合为 `I`
- 候选块集合为 `B`
- 候选层集合为 `T`

对候选块 `b in B`，定义：

- `l_b, w_b, h_b`：块的长宽高
- `q_b`：块重量
- `v_b`：块体积
- `a_ib`：块 `b` 中包含类型 `i` 的货物件数
- `stack_b`：该块是否允许承载上层

对候选层 `t in T`，定义：

- `h_t`：层高
- `q_t`：层总重量
- `v_t`：层总体积
- `a_it`：层 `t` 中包含类型 `i` 的货物件数
- `p_t`：层内二维排布方案

决策变量：

- `y_t in {0,1}`：是否选取候选层 `t`
- `lambda`：综合满载率

这组变量体现的是：

- 主模型不直接对每件货物坐标做全局优化
- 主模型只决定哪些候选层被选中
- 层内几何可行性由候选层生成阶段保证

#### 2. 目标函数

模型A不直接最大化利润，而是最大化综合满载率：

```text
max lambda
```

并约束：

```text
lambda <= (sum(v_t * y_t)) / (L * W * H_eff)
lambda <= (sum(q_t * y_t)) / Q
```

这样定义的原因是：

- 只看体积，会偏向泡货
- 只看重量，会偏向重货
- 题目要求“满容”和“满载”尽量同时达到，所以要用一个平衡两者的指标

因此，`lambda` 可以理解为：

> 体积利用率与载重利用率中的较小者。

#### 3. 约束条件

模型A主约束包括：

```text
sum(h_t * y_t) <= H_eff
sum(q_t * y_t) <= Q
sum(a_it * y_t) <= n_i,   for all i in I
```

其中：

- 第一条对应车厢有效高度限制
- 第二条对应车辆载重限制
- 第三条对应每类货物的可用数量限制

除此之外，层内几何可行性在候选层构造阶段已经保证，包括：

- 不超出车厢底面边界
- 块之间不重叠
- 定向件姿态合法
- 易碎件不承担上层载荷
- 支撑与承压规则满足

顶部安全间隙通过 `H_eff = H - 3` 统一吸收。

局部插入阶段负责进一步利用层模型未覆盖的零散空间，例如：

- 顶层支撑面上的剩余空位
- 边角碎片空间
- 可补入的少量 `G1`、`G2` 等小件

#### 4. 约束的现实含义

模型A中的每类约束都对应一个现实装载要求：

- 高度约束：不能超过车厢有效高度
- 载重约束：不能超过车辆额定载重
- 数量约束：不能虚构货物
- 层内可行性约束：保证装得进去、摆得稳、不违规
- 安全间隙约束：满足题目要求的顶部预留空间

所以模型A不是抽象的“数学拼箱”，而是贴近题意的“物理可执行装载模型”。

### 4.5 求解方法

模型A的求解方法固定按下面六步描述：

1. 货物展开与合法姿态生成
2. 候选块生成
3. 候选层生成
4. 层组合选择
5. 局部插入修补
6. 结果输出与校验

更具体地说：

- `块` 用来压缩规模
- `层` 用来承接二维排布和稳定支撑
- `层组合选择` 用来得到单车初始装载方案
- `局部插入修补` 用来吸收层模型遗漏的小件和边角空位
- `结果输出与校验` 用来生成最终坐标方案

其中“局部插入修补”必须明确说明其作用：

> 分层法容易在顶层和边角留下可利用的小空位，因此在初始分层方案得到后，需要对剩余货物逐件尝试插入，以减少明显可利用空间被浪费的情况。

`ALNS` 如果使用，可以作为**可选改进模块**，而不是模型A的唯一核心。  
当前文档推荐的主线仍然是“块化 + 分层 + 局部插入修补”。

### 4.6 输出格式

模型A最终建议输出两类结果：

#### （1）明细结果表

采用“一行一件货物”的扁平表格式，推荐字段：

- `vehicle_id`
- `item_uid`
- `item_type`
- `x, y, z`
- `dx, dy, dz`
- `orientation_code`
- `layer_id`
- `support_id`

推荐朝向编码：

- `O1 = (l, w, h)`
- `O2 = (l, h, w)`
- `O3 = (w, l, h)`
- `O4 = (w, h, l)`
- `O5 = (h, l, w)`
- `O6 = (h, w, l)`

#### （2）汇总结果表

推荐字段：

- `vehicle_volume_cm3`
- `vehicle_weight_capacity_kg`
- `used_volume_cm3`
- `used_weight_kg`
- `space_utilization`
- `weight_utilization`
- `lambda`
- `loaded_item_counts`
- `residual_counts`
- `repair_inserted_counts`
- `repair_success`

其中：

- `space_utilization = used_volume_cm3 / vehicle_volume_cm3`
- `weight_utilization = used_weight_kg / vehicle_weight_capacity_kg`
- `lambda = min(space_utilization, weight_utilization)`

## 五、模型B：问题一第二子问题

### 5.1 要解决什么

模型B对应问题一第二子问题，核心任务不是“某一辆车怎么装最好”，而是：

> 在只允许使用一种车型的前提下，把全部货物运输完成，并使总车辆数最少。

换句话说，模型B研究的是“**全局怎么分配最省车**”。

### 5.2 为什么不能直接重复模型A

模型B不能简单理解为“把模型A跑很多遍”，原因是：

1. 模型A是单车局部优化，模型B是多车全局优化。
2. 单车较好的方案重复使用后，可能会把某些货物提前用光，导致后续车辆更难装。
3. 模型B的关键决策是“不同车辆之间如何互补”，而不是“每辆车各自尽量满”。

所以模型B必须重新建模，不能直接把模型A的结果机械复制。

### 5.3 建模思路

模型B采用一种和模型A完全衔接的模式法思路：

1. 先由模型A生成单车可行装载模式。
2. 再把“一辆车的一种可行装法”整体抽象成一个装载模式。
3. 然后由模型B在模式库中选择尽量少的模式，使全部货物被刚好覆盖。
4. 最后对少量剩余货物执行残量修补，必要时再补充小模式。

因此，模型B采用：

> **装载模式集合划分模型 + 模式库生成 + 整数主问题搜索 + 残量修补**

这套方法的思想非常清楚：

- 模型A负责“造模式”
- 模型B负责“选模式”

### 5.4 主问题

设：

- `I`：货物类型集合，`i in I`
- `P`：可行装载模式集合，`p in P`
- `n_i`：第 `i` 类货物总需求件数
- `a_ip`：模式 `p` 中装入的第 `i` 类货物件数
- `y_p`：模式 `p` 被使用的次数

其中每个模式 `p` 表示：

- 一辆同型号车辆的一种完整装载方案
- 该方案已经满足尺寸、重量、堆叠、方向、稳定性等所有约束

主问题目标函数：

```text
min sum(y_p),  p in P
```

约束：

```text
sum(a_ip * y_p) = n_i,   for all i in I
y_p in Z_>=0,            for all p in P
```

这三个式子的含义是：

- 每选一次模式，就相当于用一辆车
- 所有模式中的各类货物数量加总后，必须刚好等于总需求
- 模式使用次数必须是整数，因为不存在半辆车

模型B主问题里不直接写坐标变量，是因为：

> 主问题只做全局模式选择，不再重复处理单车内部三维排布。

### 5.5 模式生成

由于可行装载模式数量很多，无法一次性完全列出，因此需要先构造一个模式库。

模式来源建议包括：

- 综合满载率较高的模式
- 重货优先模式
- 泡货优先模式
- 重泡平衡模式
- 残量修补模式

这些模式统一由模型A求解器产生。  
也就是说，模型A在这里不再只是“单车求一遍结果”，而是：

> 一个可以反复调用的单车模式生成器。

每个模式至少需要记录：

- 各类货物件数
- 单车坐标方案
- 体积利用率
- 载重利用率
- 综合满载率

### 5.6 主问题求解策略

模型B的求解过程建议按下面顺序写：

1. 固定一种车型。
2. 由模型A生成若干单车装载模式，构成初始模式库。
3. 在模式库上求解整数主问题，得到当前最少车辆组合。
4. 检查是否还有少量剩余货物未被精确覆盖。
5. 若有剩余，则调用模型A生成残量修补模式。
6. 必要时补充小模式，再重新求解主问题。
7. 输出最终车辆数与各车装载方案。

因此，模型B的实现主线应写成：

- 模式库生成
- 整数主问题搜索
- 残量修补

而不是把“列生成 + 分支定价”作为当前代码必须实现的主线。

### 5.7 模型A与模型B的关系

模型A与模型B的关系可以清楚地概括为：

- 模型A是**单车模式生成器**
- 模型B是**全局模式选择器**
- 模型A负责解决单车内部如何装得更好
- 模型B负责解决多辆车之间如何分得更省

因此，模型B并不是简单重复模型A，而是把模型A的输出进一步抽象成模式，再在模式层面做全局优化。

### 5.8 论文可直接使用的模型B表述

可直接写为：
